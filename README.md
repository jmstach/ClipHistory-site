# cliphistory.stach.uk

Landing page for [ClipHistory](https://cliphistory.stach.uk) — a super-tiny,
privacy-respecting, macOS-native clipboard manager. A personal fork of
[weiykong/ClipHistory](https://github.com/weiykong/ClipHistory).

Static HTML + one CSS file + one vanilla JS file, plus a single serverless
subscribe endpoint. No framework, no build step. Ships via Vercel from this
repo on every push to `main`. Design adapted from the Markset landing page:
Swiss baseline grid, Helvetica, black on white (inverts in dark mode).

## Files

- `index.html` — page content (static hero, demo video, features, colophon)
- `style.css` — Swiss baseline grid, Helvetica, black on white
- `subscribe.js` — reveals the email form after Download, POSTs to the API
- `favicon.svg` — paperclip mark, adapts to light/dark
- `hero.mp4` + `hero.jpg` — looping demo (bottom-tray mode) and its poster frame
- `og.png` — social card, generated by `scripts/generate-og.swift`
- `icon.png` — app icon, used by the OG generator
- `release-notes/`, `privacy/`, `terms/` — sub-pages
- `api/subscribe.ts` — **stub** endpoint (validates + acknowledges, no storage yet)

## Local preview

```bash
python3 -m http.server 4242
# open http://localhost:4242
```

The static pages work over plain HTTP. The subscribe form needs the serverless
function, so run `vercel dev` to exercise that path end-to-end.

## Subscribe endpoint

`api/subscribe.ts` is currently a stub: it validates the payload and returns
success without persisting anything. Wire up real storage (mirror Markset's
R2-backed JSON list, or another provider) before relying on captured addresses.
See the comment at the top of the file.

## Analytics

[Plausible](https://plausible.io) — privacy-friendly and cookieless. The
snippet sits in the `<head>` of every page. No PII, no cross-site tracking;
the privacy page documents it.

## OG image

```bash
swift scripts/generate-og.swift   # → og.png (1200×630)
```

Reads `icon.png` from the repo root, so run it from there.

## Pre-commit hooks

A tracked pre-commit hook in `.githooks/` runs `scripts/smarten-quotes.js` on
staged `.html` files and re-stages the result, turning straight quotes into
typographic ones. On a fresh clone, run `npm install` once — the `postinstall`
script points `core.hooksPath` at `.githooks`. Bypass on a single commit with
`git commit --no-verify`.
